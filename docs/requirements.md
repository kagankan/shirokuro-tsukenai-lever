# 要件定義

## 概要

YouTubeチャンネル「[白黒つけない会議](https://www.youtube.com/@not_shirokuro)」に登場するレバーを再現したWebアプリケーション。
あるお題に対して、参加者がそれぞれ0〜100の度合いで意見を表明する。
物理的なレバーを左右に倒す動作をWeb上で再現し、参加者全員の現在値をリアルタイムに共有・表示する。

ファンメイドの個人利用アプリ。

## 機能要件

### ルーム管理

- ルームを作成すると一意のURLが発行される
- URLを共有するだけで参加できる（ログイン不要）
- 参加者数は3人がベースだが、4人以上も参加可能（上限8人程度。画面レイアウトの都合）
- 参加時にニックネームとアイコン（プリセットから選択）を設定する

### お題管理

- ルーム内の全員がお題を入力・変更できる
- お題の変更はリアルタイムに全員へ反映される
- お題変更時にコンフリクトが起きないよう、最後の入力が優先される（Last Write Wins）

### レバー操作

- 各参加者は自分のレバーを操作できる
- レバーは0（左端）〜100（右端）の値をとる
- 物理レバーを横から見たような円弧の軌道で動作する
- 操作はドラッグまたはタッチで行う
- レバーの現在値はリアルタイムに全参加者の画面へ反映される

### 表示

- スマホ縦長レイアウトを基本とする
- 画面は上から3つのエリアに分かれる:
  1. **お題エリア（上部）**: 現在のお題の表示・編集
  2. **結果マップ（中央）**: アーチ状のマップ上に、全員のアイコンと数値が配置される。0〜100の値に応じてアーチ上の左右位置が決まる
  3. **自分のレバー（下部）**: 自分が操作するレバーUI

### リアルタイム性

- レバー操作の反映は低遅延（体感でほぼリアルタイム）
- Supabase RealtimeのBroadcast機能を使用し、レバー位置の変更をpublish/subscribeする
- レバー位置は高頻度で変わるため、DB書き込みは間引く or 最終値のみ永続化する

## マイルストーン

### M1: リアルタイム同期（Broadcastのみ）

- ルーム作成・参加・お題設定の基本機能
- レバー操作のリアルタイム同期（Broadcastのみ、永続化なし）
- ページリロードするとレバー値はリセットされる

### M2: レバー値の永続化

- レバー値をDBに保存し、リロードしても復元できるようにする
- 操作中は高頻度のためBroadcastで同期し、操作完了時にDBへ書き込む

### 認証・セキュリティ

- ユーザー認証は設けない
- SupabaseのAnon Keyを使用（RLSは全操作許可）
- ルームURLを知っている人のみがアクセス可能（URLの推測困難性で担保）

#### フロントエンドから直接DB操作するリスクについて

Anon Keyは公開情報であり、知識のある人なら直接APIを叩いてDB操作が可能。以下の理由からこのリスクを許容する:

- **UUIDの推測困難性**: ルームIDはUUID v4のため、他人のルームを総当たりで特定するのは現実的に不可能
- **データの機密性がない**: 個人利用のファンメイドアプリであり、お題やレバー値を改ざんされても実害がない
- **手軽さを優先**: ログイン不要で即座に使えることが要件。認証を入れると利便性が大きく下がる

将来的に必要になれば、Supabaseの匿名認証（Anonymous Sign-In）を導入してRLSを厳密化できる。

### デプロイ

- GitHub Pagesで静的サイトとしてホスティング
- フロントエンドからSupabase APIに直接接続（サーバーサイド不要）
- ルーティングライブラリは使用しない。`?room={id}` の有無で画面を分岐する
